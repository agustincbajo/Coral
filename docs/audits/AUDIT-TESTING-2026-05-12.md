# Audit: Testing — Coral v0.34.1
Date: 2026-05-12
Auditor: claude (sonnet)
Scope: workspace-wide (10 crates), CI gates (`.github/workflows/ci.yml`, `nightly.yml`, `playwright-ci.yml.disabled`), prior audit raw at `audit/raw/test-quality.md`. Excluded: SPA-internal JS unit tests (no harness present), release-binary smoke on real macOS/Linux VMs (BACKLOG #3, no host available).

## Executive summary

Coral's deterministic test suite (1810 passing) is dense at the unit layer of `coral-core`, `coral-stats`, and `coral-lint`, and is well-anchored at the BC + snapshot boundary (`bc_regression.rs`, 19 `insta` snapshots with rigorous SHA/timestamp/tempdir filters). The pyramid is, however, lopsided: integration and snapshot coverage track only the CLI surface; the `coral-ui` REST surface and the `coral-test` orchestrator have **zero out-of-process tests** despite being publicly exposed user paths in v0.32+. There is **no fuzzing** (`cargo-fuzz` not wired) and **no mutation testing in CI** despite the `coral test mutants` wrapper already shipping. Property tests (8 files) cover `coral-core` parsers only; the lint engine has a single property test and the markdown/wikilink corpus has happy-path-biased strategies (per the v0.30.0 audit F4).

Top three actions, in priority order: (1) close the loopback-auth coverage gap in `coral-ui/src/auth.rs` — three security-critical public functions (`validate_host`, `validate_origin`, `require_bearer`) have **no tests** beyond their helpers; (2) land Playwright behind the `workflow_dispatch` trigger described in BACKLOG #4 so the 14 specs stop bit-rotting; (3) introduce a `cargo-fuzz` harness for `frontmatter::parse` + `wikilinks::extract` + `config::Config::from_str` and run a 4-hour soak nightly.

The 84 / 14 / 2 split (unit / integration / E2E) is healthy in shape — closer to the inverted-pyramid risk than the ice-cream cone risk. Two structural smells: (a) `coral-ui` and `coral-stats` have *zero integration tests* (unit-only); (b) `coral-cli` integration tests dominate the integration layer (25 of 42 files) so any cross-crate behavior bug surfaces only when the binary is involved. The 19-snapshot `insta` suite covers CLI markdown output exclusively — REST API JSON envelopes, MCP JSON-RPC bodies, and the `coral self-check` envelope have no snapshot pins.

Recommended targets: 75 / 18 / 7 (unit / integration / E2E) with the integration shift coming from `coral-ui::tests/` (TEST-11) and `coral-mcp::tests/mcp_stdio_smoke.rs` (TEST-07). Fuzz layer target: 4 active targets running nightly.

## Methodology

1. Read `Cargo.toml`, `BACKLOG.md`, and `.github/workflows/*.yml` to fix the test inventory and CI gates.
2. Enumerated `#[test]` and `#[cfg(test)]` annotations across all `crates/*/src/**/*.rs` and `crates/*/tests/**/*.rs` (output: 1 912 raw `^\s*#\[test\]` matches across 217 files).
3. Cross-referenced the prior multi-agent audit at `audit/raw/test-quality.md` (agent aa229f15ef5ae6fd9) and `audit/SUMMARY.md` to avoid re-finding closed items. Findings F2, F4, F8, F9 of that audit were re-checked against current code; F4 (slug strategy bias) confirmed still open and lifted into TEST-09.
4. Looked at file-level test density (tests per LoC) instead of running `cargo llvm-cov` locally (not installed; CI run takes ~30 min serial). Density is a *proxy* — not a substitute — for measured line coverage. Where density is misleading (e.g. one mega-test that exercises 80% of a file), the finding text calls out the qualitative gap.
5. Manually read the security-critical surfaces: `coral-ui/src/auth.rs`, `coral-mcp/src/transport/stdio.rs`, `coral-session/src/distill_patch.rs`, `coral-cli/src/commands/self_check.rs`. The auth gap (TEST-01) and stdio shim gap (TEST-07) emerged from reading those files end-to-end.
6. Verified the Playwright suite count and the seed-fixture proposal against `.github/workflows/playwright-ci.yml.disabled` line by line. No tests were run; no fixtures generated.

## Scope NOT audited

- **`coral-ui/assets/src/`** TypeScript / Svelte components: no JS test harness present (no `jest`/`vitest`/`playwright unit` config), so unit-test density at the SPA layer is uninspected. Playwright E2E is the only proposed coverage and it is itself disabled (TEST-05).
- **Real-binary smoke on macOS/Linux VMs** (BACKLOG #3): no host available. The cross-platform CI smoke at `.github/workflows/ci.yml:118-156` only runs `coral --version` + `coral init` against ephemeral GitHub-hosted runners.
- **Live MCP client interop**: no `claude-code` install in CI; the MCP HTTP/SSE tests use raw curl-shaped envelopes. A real Claude Code session is not exercised on any CI tier.
- **`coral monitor up` long-running loop**: integration test (`monitor_e2e.rs`, 188 LoC) covers one cycle only; no soak test of the daemon over hours.
- **Memory leak / valgrind-equivalent**: no `cargo-careful` or `cargo-miri` job is configured. The mutation-testing wrapper (TEST-06) is the closest substitute and is itself not in CI.
- **JS-side bundle-size / a11y / Lighthouse regression**: no `ui-build.yml` step measures the SPA bundle size or runs `lighthouse-ci`. The SPA ships embedded at compile time (`include_dir`), so a 2× regression would not break the build but would inflate `coral` binary size invisibly.

- **TEST-M1 (Medium):** `assert!(result.is_ok())` swallows error context at 9 sites surfaced by audit F2 (`audit/raw/test-quality.md:3-9`). Pattern still present — e.g. `crates/coral-env/src/healthcheck.rs:126`, `crates/coral-cli/src/commands/project/doctor.rs:233`. Replace with `.expect("descriptive message")` or `let _ = result.unwrap();`.
- **TEST-M2 (Medium):** `coral-cli::commands::doctor` (520 LoC) has 6 inline tests; the FR-ONB-* wizard's four provider mini-paths are flagged in BACKLOG #5 as the v0.36.0 ratchet target. Pre-emptively land branch-level tests on the wizard before bumping the floor.

Windows ratchet is at 600 ms with a measured local max of 394 ms — a 50%+ slowdown would be caught but a 30 % slowdown lands silently. Consider adding a *mean* check at 450 ms alongside the max check.
- **TEST-M4 (Medium):** `coral test mutants` wrapper ships (`crates/coral-cli/src/commands/mutants.rs`) with 12 unit tests but is uncalled in any workflow. See TEST-06.
- **TEST-M5 (Medium):** Snapshot tests at `crates/coral-cli/tests/snapshot_cli.rs` filter ANSI, ISO-8601, SHA, tempdir paths (`:183-:224`) but do not filter `cargo` toolchain version or `rustc`-emitted error messages. - **TEST-M6 (Medium):** `crates/coral-cli/tests/openapi_adversarial.rs:172` 33-MiB cap test asserts only `cases.is_empty()` (audit F10). Should additionally assert which rejection-stage fired (size cap vs deserialize). Open in prior audit, still applicable.
